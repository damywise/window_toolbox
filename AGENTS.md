# window_toolbox — Agent Guide

## What this project is

`window_toolbox` (v0.0.7) is a Flutter FFI plugin package for fully custom window chrome and platform-window utilities on desktop. It gives Flutter applications the ability to replace the native titlebar with custom widgets (drag areas, minimize/maximize/close buttons, traffic lights), manage window frames, handle window lifecycle callbacks (close, resize, fullscreen), and access platform-specific window properties (e.g., `NSWindowCollectionBehavior` on macOS, `WindowStateLinux` bitmask, Win32 `canMinimize`/`canMaximize`).

Target platforms: Windows, macOS, Linux (desktop only).

## Tech stack & requirements

- **Language:** Dart 3.12+ / Flutter SDK >=3.42.0 (pinned via `.fvmrc` to commit `67dec4430000fb69cc5b8ab8c998f9c2bcc588da`).
- **FFI bindings:** Generated by `package:ffigen` from C (Linux, `src/linux.h`) and Objective-C (macOS, `src/macos.h`) headers.
- **Native build:** `package:native_toolchain_ninja` + `package:hooks` + `package:code_assets` — native `.c`/`.m` files compile automatically during `flutter build`.
- **Win32:** Uses `package:win32` directly (no native compilation needed).
- **`headless_widgets`:** Direct runtime dependency (`^0.0.7`); provides the `Button` widget used by `MinimizeButton`, `MaximizeButton`, and `CloseButton`.
- **`ffi` / `logging`:** Runtime dependencies used by the FFI layer.
- **Package type:** `package_ffi` (per `.metadata`).
- **Lint:** Inherits `package:flutter_lints/flutter.yaml`; ignores `implementation_imports` and `invalid_use_of_internal_member`; formatter set to `trailing_commas: preserve`.

## Public API surface

All public symbols are exported from `lib/window_toolbox.dart` via four re-exports. Importing `package:window_toolbox/window_toolbox.dart` provides:

The `enableCustomWindow()` and `configureFramelessWindow()` methods are provided by the `CustomWindowExtension` on `BaseWindowController`.

### Widgets

Defined in `lib/src/widgets.dart`. All widgets silently no-op when `enableCustomWindow()` has not been called on the controller.

| Widget | Purpose |
|--------|---------|
| `WindowDragArea` | Makes its child a draggable titlebar region. |
| `WindowDragExcludeArea` | Nested inside `WindowDragArea`; marks sub-region as non-draggable (for buttons). |
| `WindowTrafficLight` | Positions macOS traffic-light buttons. Mode: `visible`, `invisible`, `removed`. |
| `WindowTrafficLightInactiveConfigration` | Configures inactive appearance of traffic lights (background, border, etc.). |
| `MaximizeButton` | Builder: `(BuildContext, TitlebarButtonState, bool isMaximized)`. On Win32, hover triggers snap-layout flyout. |
| `MinimizeButton` | Builder: `(BuildContext, TitlebarButtonState)`. |
| `CloseButton` | Builder: `(BuildContext, TitlebarButtonState)`. Triggers `onWindowCloseRequested`. |
| `WindowBorder` | Shadow / border / corner clipping. Required on Linux; no-op on Win32/macOS. |
| `TitlebarButtonState` | `{bool enabled, bool hovered, bool pressed}`. |
| `WindowTrafficLightMode` | Enum: `visible`, `invisible`, `removed`. Used by `WindowTrafficLight`. |

### Platform delegates

Mixins that receive window lifecycle callbacks. Attach via `controller.addDelegate(delegate)` using the platform extension.

| Mixin | Callbacks |
|-------|-----------|
| `WindowDelegateWin32` | `windowWillClose`, `windowWillResizeToSize`, `enterSizeMove`, `exitSizeMove` |
| `WindowDelegateMacOS` | `windowWillClose`, `windowWillResizeToSize`, `windowWillStartLiveResize`, `windowDidEndLiveResize`, `windowWillUseStandardFrame`, fullscreen enter/exit (4 hooks) |
| `WindowDelegateLinux` | `windowWillClose`, `windowStateDidChange` |

### Platform controller extensions

Extensions on Flutter's internal `BaseWindowController` subclasses, defined in the respective `lib/src/<platform>_extra.dart` file.

| Extension | Properties / Methods |
|-----------|----------------------|
| `WindowControllerWin32Extension` | `addDelegate`, `removeDelegate`, `addWindowsMessageHandler`, `removeWindowsMessageHandler`, `canMinimize` (get/set), `canMaximize` (get/set), `getWindowFrame`, `setWindowFrame`, `updateSize` |
| `WindowControllerMacOSExtension` | `addDelegate`, `removeDelegate`, `canMinimize` (get/set), `canClose` (get/set), `collectionBehavior` (get/set, uses `NSWindowCollectionBehavior`), `getWindowFrame`, `setWindowFrame`, `updateSize` |
| `WindowControllerLinuxExtension` | `addDelegate`, `removeDelegate`, `getWindowState` → `WindowStateLinux` |

Additional enums and types:

- `NSWindowCollectionBehavior` — bitmask with values: `defaultBehavior`, `canJoinAllSpaces`, `moveToActiveSpace`, `managed`, `transient`, `stationary`, `participatesInCycle`, `ignoresCycle`, `fullScreenPrimary`, `fullScreenAuxiliary`, `fullScreenNone`, `fullScreenAllowsTiling`, `fullScreenDisallowsTiling`, `primary`, `auxiliary`, `canJoinAllApplications`.
- `WindowStateLinux` — decoded bitmask booleans: `withdrawn`, `iconified`, `maximized`, `sticky`, `fullscreen`, `above`, `below`, `focused`, `topTiled`, `topResizable`, `rightTiled`, `rightResizable`, `bottomTiled`, `bottomResizable`, `leftTiled`, `leftResizable`.
- `Win32MessageHandler` — `int? Function(HWND, int message, int wParam, int lParam)`.

## Typical usage

```dart
final controller = RegularWindowController(...);
controller.enableCustomWindow();

runApp(MaterialApp(
  home: Scaffold(
    body: Column(children: [
      WindowDragArea(child: Row(children: [
        if (Platform.isMacOS) const WindowTrafficLight(),
        const Spacer(),
        MinimizeButton(builder: (c, s) => /* icon */),
        MaximizeButton(builder: (c, s, isMax) => /* icon */),
        CloseButton(builder: (c, s) => /* icon */),
      ])),
      const Expanded(child: YourContent()),
    ]),
  ),
));
```

On Linux, wrap the body with `WindowBorder(cornerRadius: 12, child: ...)` for shadow and rounded corners (no system compositor).

## Build, test, and codegen

- **Analyze:** `fvm flutter analyze` (or `dart analyze`).
- **Format:** `dart format .` (trailing commas preserved per `analysis_options.yaml`).
- **Test:** `fvm flutter test`.
- **Run example:** `cd example && fvm flutter run -d <windows|macos|linux>`.
- **Regenerate FFI bindings:** `dart run tool/ffigen.dart` (run on macOS or Linux; Windows does not need ffigen).
- **Native build:** Automatic via `hook/build.dart` during `flutter build` — no manual compilation step needed.

## Hard rules (do not violate)

- DO NOT edit `lib/src/macos.g.dart` or `lib/src/linux.g.dart`. These are generated by ffigen. If a C/Obj-C declaration in `src/` changes, update the corresponding `.h` file and rerun `dart run tool/ffigen.dart`.
- DO NOT import from `lib/src/` files directly from outside the package. Always use the barrel: `import 'package:window_toolbox/window_toolbox.dart';`.
- DO NOT add new public symbols without re-exporting them from `lib/window_toolbox.dart`.
- DO NOT forget to call `controller.enableCustomWindow()` — widgets silently no-op without it.
- DO NOT modify `hook/build.dart` unless changing the native build pipeline. Windows builds do not need it at all.
- DO NOT add new dependencies without updating `pubspec.yaml` and committing `pubspec.lock`.

## Adding new functionality

- **New widget:** Add to `lib/src/widgets.dart`. Keep state classes private (`_*State`). If the state needs to communicate with the native window, obtain the `CustomWindow` instance via `CustomWindow.forController()` from `lib/src/custom_window.dart`.
- **New platform-specific API:** Add a method to the appropriate extension in `lib/src/<platform>_extra.dart`. If a native call is required, add the C/Obj-C function in `src/<platform>.{c,m,h}`, then regenerate bindings with `dart run tool/ffigen.dart`.
- **New delegate callback:** Add the method to the relevant `WindowDelegate*` mixin, implement the hook in `custom_window_<platform>.dart`, and wire it through the native delegate config struct (`cw_delegate_config_t`).
- **New platform:** Create `custom_window_<platform>.dart`, the `<platform>_extra.dart` extension, and the corresponding native source files. Wire the factory in `custom_window.dart`. Update `hook/build.dart` if a new compiler toolchain is needed. Add re-exports to the barrel.
